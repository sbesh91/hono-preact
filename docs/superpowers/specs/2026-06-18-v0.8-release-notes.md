# hono-preact v0.8.0: release notes

**Released:** 2026-06-18

v0.8.0 replaces the `Persist` / `PersistHost` pattern with a first-class primitive for persistent, live UI: a `live` loader on a layout's server module. A layout child that consumes the loader with `loader.useStream(...)` stays mounted and connected across every navigation within that layout's scope, with no manual DOM teleportation and no separate lifecycle to manage.

## Highlights

### `defineLoader({ live: true })`: long-lived client subscriptions

Marking a loader as `live` opts it into a distinct mode: it is never invoked during SSR (so an infinite generator cannot hold the document response open), its timeout defaults to `false` (no 30-second cap), and it can only be consumed via `loader.useStream(...)`. The stream connects once when the owning layout mounts and is aborted when the layout unmounts.

```ts
// src/pages/projects/layout.server.ts
import { defineLoader, type LoaderCtx } from 'hono-preact';
import type { ActivityEvent } from './activity-stream.js';

export const serverLoaders = {
  activity: defineLoader(
    async function* (ctx: LoaderCtx): AsyncGenerator<ActivityEvent> {
      for await (const event of subscribeActivity(ctx.signal)) yield event;
    },
    { live: true }
  ),
};
```

### `loader.useStream(opts)`: accumulate a live stream

`loader.useStream({ reduce, initial })` subscribes to a live loader and folds every incoming chunk into accumulated state. It returns `{ data, status, error }`. During SSR and before hydration, `data` is the `initial` value and `status` is `'connecting'`:

```tsx
// src/pages/projects/layout.tsx
import { serverLoaders } from './layout.server.js';
import type { ActivityEvent } from './activity-stream.js';

const activityLoader = serverLoaders.activity;

function ActivityBar() {
  const { data: events, status } = activityLoader.useStream<ActivityEvent[]>({
    reduce: (acc, e) => [e, ...acc].slice(0, 50),
    initial: [],
  });

  return (
    <ul>
      {events.map((e) => (
        <li key={e.id}>{e.actor}: {e.taskTitle}</li>
      ))}
    </ul>
  );
}
```

The connection survives navigating between child routes of the layout (e.g. `/projects/alpha` to `/projects/beta`) because the layout stays mounted. It closes when the user leaves the layout's scope entirely.

### Persistent UI via layout scoping

Persistent UI is now expressed purely as layout structure: whatever the layout renders is persistent across intra-scope navigation. No wrapper component, no registry, no DOM teleportation. Scope the layout to the routes the UI should survive:

- Root layout (`path: '*'` children): the UI persists across the whole app.
- Prefix layout (e.g. `path: '/projects'`): the UI persists within `/projects/**`.

See [Live Loaders](/docs/live-loaders) for the full guide and options reference.

## Breaking changes

### `Persist`, `PersistHost`, and `PersistProps` removed

The `Persist` component, `PersistHost` component, and `PersistProps` type are removed from `hono-preact`. Any import of these names is a compile error after upgrading.

**Migration:** render the component that previously lived inside `<Persist>` directly as a child of the layout that covers the routes it should survive. Use `defineLoader({ live: true })` and `loader.useStream(...)` if the component needs a long-lived data connection. No wrapper, no `id` prop, no `<PersistHost>` placement.

## New exports

```ts
import { defineLoader } from 'hono-preact';
import type { StreamStatus, UseStreamOptions, UseStreamResult } from 'hono-preact';
```

`defineLoader` is unchanged except for the new `live` option on its second argument. `StreamStatus`, `UseStreamOptions`, and `UseStreamResult` are type-only exports for annotating `useStream` consumers.

## Upgrading

```bash
npm install hono-preact@0.8.0
```

Scaffold a new app with the matching CLI:

```bash
npm create hono-preact@0.8.0
```

`hono-preact-ui` is unchanged in this release and keeps its own version line.
