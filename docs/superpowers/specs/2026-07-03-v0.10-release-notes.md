# v0.10 release notes (draft)

Date: 2026-07-03
Status: draft for the v0.10 cut. Consolidates the two standalone note fragments
(`2026-06-27-errorboundary-loading-release-note.md`,
`2026-06-28-realtime-trap-enforcement-release-note.md`) and adds the breaking
changes those did not cover. Review against the shipped code before publishing.

The headline of v0.10 is the **loader inversion**: route-binding becomes a
composition (`serverRoute(r).loader`), bare `defineLoader` becomes the
route-independent primitive, and streaming is expressed as a loader rather than a
separate primitive. Several smaller boundary disciplines ship alongside it.

## Breaking changes

### 1. Loader inversion: route-binding by composition

Server loaders are now bound to a route by composition instead of by colocation
alone:

- **`serverRoute('/items/:id').loader(fn)`** — route-bound. The ctx carries a
  typed `location` (`pathParams`/`searchParams`), the loader inherits the route
  node's `use` chain (the page/auth tier), and it participates in SSR.
- **`defineLoader(fn)`** — route-independent. The ctx is `{ c, signal, call }`
  with **no `location`**, the loader runs with the app-level `use` chain only,
  and it is consumable anywhere (including outside the router).

**Migration:**

- **`LoaderCtx` has no `location` by default.** A bare `LoaderCtx` (no generic)
  is the standalone shape and does not expose `location`. A loader that reads
  `ctx.location` must be route-bound: author it with `serverRoute(r).loader(fn)`
  (which infers the located ctx) or annotate the ctx explicitly as
  `LoaderCtx<RouteParams<'/items/:id'>>`. A bare `defineLoader` that reads
  `ctx.location` no longer type-checks.
- **Route-node `use` guards apply to the loader RPC only for a route-bound
  loader.** In v0.9 every loader inherited the current route's `use` gates on the
  loaders RPC (resolved from the request path). In v0.10, **only
  `serverRoute(r).loader` inherits the route node's guards; a bare
  `defineLoader` colocated on a guarded route does not.** Auth is now
  composition-evident: `serverRoute` gives you route guards, bare `defineLoader`
  does not. **Migrate any loader that must be protected by a route-level guard to
  `serverRoute('<route>').loader(...)`.** This is a silent change on upgrade
  (the bare form still compiles), so audit colocated loaders on guarded routes.

  > Release-blocking follow-up (tracked separately): the framework should emit a
  > boot/build diagnostic when a bare loader is colocated on a route that carries
  > `use`, and the docs demo (`apps/site/.../projects-shell.server.ts`) should be
  > migrated to `serverRoute('/demo/projects').loader(...)` before this ships, so
  > the guarded-route example is not a copy-paste auth footgun.

- **`LiveLoaderOptions` is no longer exported.** The live-stream option surface
  moved onto the loader/`liveStream` composition; drop any import of the type.

### 2. `LoaderState` / `StreamState` cold arm carries `data?: never`

The cold `loading` arm of `LoaderState<T>` and the cold `connecting` arm of
`StreamState<T>` now declare `data?: never` (instead of omitting `data`). This
makes `data` uniformly readable as `T | undefined` on the bare union without a
narrow, but it **breaks `in`-based narrowing**: `'data' in state` now returns
`true` on every arm, including the cold arm that carries no value.

**Migration:** narrow on `status` (`state.status === 'success'`), or use a
structural data-bearing check, instead of `'data' in state`. Read `data` only
after narrowing to a data-bearing status.

### 3. Realtime cross-runtime trap enforcement

(Full text: `2026-06-28-realtime-trap-enforcement-release-note.md`.)

- **`socket.data` / `conn.data` are now `Readonly`.** On a hibernatable
  Cloudflare Durable Object an in-place mutation of the per-connection bag
  silently vanishes; the read-only type turns that into a compile error.
  Migration: capture a closure variable in `open()` (sockets) or `onJoin()`
  (rooms) for mutable Node-only state, or use `setPresence` (rooms) for state
  that must evolve and broadcast.
- **A factory-less room's `conn.data` is now `undefined` (was `{}`)**, matching
  `socket.data` and the Cloudflare resolution path; the `defineRoom` `Data`
  generic defaults to `undefined`. Migration: add a `data` factory or guard the
  access.
- Not breaking: the Node dev server now warns when a `data` factory result
  exceeds the 6KB forward-header budget that throws at connect time on
  Cloudflare.

### 4. Hold-alive guarded navigation (advanced-consumer contract)

(Full text: `2026-06-27-errorboundary-loading-release-note.md`.)

Behavior change, not a public API change: on a client navigation between routes
that have page-layer middleware (`use`), the previous route now stays visible
while the next route's middleware chain resolves, instead of blanking out. Apps
using the normal `Routes` mounter need no migration. The one contract change for
advanced consumers: the page-middleware host now relies on the surrounding
`Router` (provided by `Routes`) as its suspense boundary rather than carrying its
own, so code that mounts the host directly via `hono-preact/internal` without a
`Router` ancestor must add one.

## New (additive)

- **Server-side caller.** `createCaller(c)` builds an HTTP-free caller bound to a
  Hono `Context` for tests and server code; `ctx.call(ref, ...)` reaches a
  loader/action in-process, running the callee's own `use` (not the app or route
  tiers), returning the authored type (no `Serialize` round-trip). Public types:
  `ServerCaller`, `CallResult<T>`, `CallLoaderLocation`.
- **Typed deny codes.** `deny(code)` accepts a named `DenyCode`
  (`BAD_REQUEST`, `UNAUTHORIZED`, `FORBIDDEN`, `NOT_FOUND`, `CONFLICT`,
  `UNPROCESSABLE`, `TOO_MANY_REQUESTS`, `INTERNAL`) that decorates the numeric
  status, so clients can `switch` on a typed code instead of sniffing the message.
  `status` stays authoritative. New exports: `DenyCode`, `DENY_CODE_STATUS`.
- **`liveStream({ topic, load })`.** Compose a route-independent live loader:
  `route.loader(liveStream({ topic, load }))` or a bare
  `defineLoader(liveStream(...))`.
- **Inference helpers.** `InferActionPayload`, `InferActionResult`,
  `InferActionChunk`, `InferLoaderData` over action/loader refs.
- **Opt out of the nav view transition (#165).** `skipNextNavTransition()`, the
  `<NavLink transition={false}>` prop, and `useNavigate`'s `{ transition: false }`
  option suppress the view transition for the next navigation.
- **Client-side schema pre-validation for `useAction` (#154).** When a `schema`
  is supplied, `useAction` validates the payload on the client before dispatch;
  the rejection shape matches the server's deny. The server still validates, so
  this is an ergonomic short-circuit, not a security boundary.
- **Colocated server-module auto-discovery + `src/server` registry
  (#215/#216/#217).** A `.server.*` file next to a route's inline
  `view`/`layout` import is discovered and wired automatically, so an explicit
  `server:` route field becomes an advanced override (non-sibling module,
  `server: false` opt-out, or runtime-generated route tables). Route-less server
  modules can live under `src/server/` at any depth.
