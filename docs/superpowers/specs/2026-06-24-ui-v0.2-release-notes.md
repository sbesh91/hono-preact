# hono-preact-ui v0.2.0: release notes

**Released:** 2026-06-24

A purely additive release: it adds the headless **Toast** primitive and changes
nothing else. The public-export surface was diffed against the
`hono-preact-ui@0.1.0` tag from a fresh build of both trees: 19 exports added,
**none removed or renamed**, and no existing component or hook was modified.

## New: Toast primitive

A Sonner-shaped notification system in the same unstyled, `data-*`-contract,
`render`-prop style as the rest of the library:

- **Imperative `toast()`** with `toast.success` / `toast.error` /
  `toast.promise` / `toast.dismiss` variants. Calls from anywhere, no context
  needed at the call site.
- **`<Toaster>` region** rendered into the top layer via the Popover API, so
  toasts float above the rest of the page without z-index wrangling, and stay
  reachable for keyboard and assistive tech.
- **Composable parts** (`ToastRoot`, `ToastTitle`, `ToastDescription`,
  `ToastClose`, `ToastAction`), each headless and fully stylable through its
  `render` prop and a `data-*` state contract, consistent with the other
  components.
- **Interaction and lifecycle:** swipe-to-dismiss, pause-on-hover and
  pause-on-window-blur auto-dismiss timers, stacking with a configurable visible
  limit, and configurable placement (`ToastPosition`).
- **Accessibility:** a polite/assertive `aria-live` announcer so toasts are read
  out without stealing focus, plus a hotkey to jump focus to the region.

See the Toast guide and full API reference at
[framework.sbesh.com](https://framework.sbesh.com).

## Breaking changes

None. This is a backward-compatible addition over `0.1.0`.

## New exports

- **Imperative API:** `toast`.
- **Region:** `Toaster` (`ToasterProps`).
- **Parts:** `ToastRoot` (`ToastRootProps`), `ToastTitle` (`ToastTitleProps`),
  `ToastDescription` (`ToastDescriptionProps`), `ToastClose` (`ToastCloseProps`),
  `ToastAction` (`ToastActionProps`).
- **Types:** `Toast`, `ToastOptions`, `ToastPosition`, `ToastRecord`,
  `ToastType`, `ToastActionData`.

## Versioning

`hono-preact-ui` versions independently of the `hono-preact` framework (this
release ships alongside framework `v0.8.0` but on its own line and tag
namespace). Still pre-1.0: stable enough to build on, may evolve ahead of 1.0.

## Install

```bash
npm install hono-preact-ui
```

Requires `preact` (>=10.11.0) as a peer dependency.
