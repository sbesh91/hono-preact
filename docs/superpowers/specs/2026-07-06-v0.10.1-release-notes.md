# hono-preact v0.10.1: release notes

**Released:** 2026-07-06

A dev-server bugfix. No API changes, and no change to how built apps behave.

## `vite dev` no longer 500s on serverless routes (Vite 8)

On Vite 8, `vite dev` returned a 500 for the route table (`/src/routes.ts`)
for any app with a route whose `view`/`layout` is a bare lazy import and has
**no** colocated `.server.*` module, which is the common case: a static home
page, a not-found route, any view without a loader. The browser reported it as
a blocked module with a disallowed MIME type (`NS_ERROR_CORRUPTED_CONTENT` in
Firefox), and the page never hydrated. The server-rendered HTML still appeared,
so the page looked half-working.

The cause was an interaction with a Vite 8 change. Route auto-discovery watched
the *absent* colocated `.server` candidate so that creating the file later would
wire up its loaders without a dev restart. In Vite 8, `addWatchFile` also
registers the watched path as a module import that import-analysis must resolve,
so watching a not-yet-existent server file made the route table fail to resolve
it.

Discovery no longer watches through the import graph. Creating or removing a
`.server.*` file while the dev server is running still re-wires the affected
routes (now through the dev server's own file watcher), so that DX is unchanged.

This only affected `vite dev`; `vite build` was never impacted.

## Upgrading

`hono-preact@0.10.1` is a drop-in replacement for `0.10.0`. There are no code
changes to make. `pnpm update hono-preact`, or a fresh `create-hono-preact`
scaffold (which pins `^0.10.0`), picks up the fix. `hono-preact-ui` is unchanged
at `0.2.0`.
