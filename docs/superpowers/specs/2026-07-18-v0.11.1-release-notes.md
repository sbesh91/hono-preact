# hono-preact v0.11.1: release notes

**Released:** 2026-07-18

An SSR bugfix. No API changes, and no change to how you write an app.

## SSR now emits a single, correct `<title>`

A page that set its own title with `useTitle` (or `useTitleTemplate`) while its
`Layout` used the `<Head>` component shipped **two** `<title>` elements in the
server-rendered HTML: the static one `<Head>` always renders, plus the injected
one carrying the page's resolved title. Because browsers use the **first**
`<title>` element in the document, the static fallback shadowed the real title,
so the pre-hydration HTML that crawlers and social-card scrapers read carried
the wrong title (for example a not-found page served the site default instead of
"Page not found"). The tab only corrected itself once client hydration ran. This
affected every `<Head>`-based app, including a fresh `create-hono-preact`
scaffold.

`renderPage` now removes the `Layout`'s own `<title>` before injecting the
resolved one, so exactly one `<title>` ships and it carries the page's title.
The removal is scoped to the document `<head>`, so a `<title>` inside a body
inline-SVG (a legitimate accessible name) is left untouched. When a page sets no
title, the `<Head defaultTitle="...">` fallback is preserved unchanged.

## Upgrading

`hono-preact@0.11.1` is a drop-in replacement for `0.11.0`. There are no code
changes to make. `pnpm update hono-preact`, or a fresh `create-hono-preact`
scaffold (which pins `^0.11.0`), picks up the fix. `hono-preact-ui` is unchanged
at `0.3.0`.
