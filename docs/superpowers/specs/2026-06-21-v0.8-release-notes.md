# hono-preact v0.8.0: release notes

**Status:** Final. The public-export-surface was diffed against the `v0.7.0` tag
on 2026-06-24 from a fresh build of both trees (a stale `dist/` would have lied),
and every diff-invisible breaking claim below was checked against `v0.7.0` source.
The lists reflect that diff plus the manually-tracked diff-invisible changes.

> **Note on the breaking-change list.** A public-export-surface diff vs `v0.7.0`
> catches *removed/renamed exports* but **misses** breaking changes that are (a)
> the removal of a *member* from an existing exported type, (b) a change to an
> existing export's *type shape*, or (c) a runtime/`engines` requirement. Those
> are recorded manually below (marked ⚠ diff-invisible) so they are not lost; it
> is the same trap that hid the Babel 8 engines bump.

## Breaking changes

- **`ActionStub` renamed to `ActionRef`** (RPC naming unification, #175). The
  client-side server-binding handle for actions now matches the `*Ref` suffix
  shared by `LoaderRef`/`SocketRef`/`RoomRef`; `defineAction` returns an
  `ActionRef`, and `useAction`/`<Form>`/`useActionResult`/`useFormStatus` accept
  it. No back-compat alias: change `import type { ActionStub }` to `ActionRef`.
- **`RouteServer` renamed to `RouteBinder`** (RPC naming unification, #175). The
  `serverRoute()` return interface (carrying `.loader`/`.liveLoader`/`.socket`/
  `.room`) is renamed to break the `RouteServer`/`ServerRoute`/`serverRoute`
  collision triad. It is almost always inferred from `serverRoute()`, so most
  code needs no change; update any explicit `import type { RouteServer }`.
- **`Persist` component removed; persistence is now by-layout** (#133). The
  `PersistHost` and `PersistProps` types are removed alongside it. Live streams
  are consumed through `loader.View`'s accumulating form.
- **Server public API narrowed** (#131). `hono-preact/server` now exports only
  `renderPage` + `HonoContext`/`useHonoContext`. The request handlers
  (`loadersHandler`/`pageActionHandler`) and their option types
  (`LoadersHandlerOptions`/`PageActionHandlerOptions`/`ActionEntry`) are now
  internal, reached by the generated entry via the private `createServerEntry`.
- ⚠ **`LoaderCache.wrap()` removed** (#142). The method was dropped from the
  publicly re-exported `LoaderCache` type (it had no callers and was vestigial).
  Diff-invisible: the `LoaderCache` type name persists, so an export-surface diff
  does not flag it.
- ⚠ **Loader/action data is now typed `Serialize<T>` at the client boundary**
  (#132, #134). `loader.useData()`, `useAction().data`, `useActionResult()`,
  `<Form onSuccess>`, and streaming `onChunk` now reflect the JSON wire shape
  (e.g. a `Date` field arrives as a `string`) instead of the server-side `T`.
  Diff-invisible: the affected exports keep their names; only their result types
  changed. A new `Serialize<T>` type is exported.
- ⚠ **`RouteParams` no longer claims a param for non-`[A-Za-z0-9_]` names**
  (#130). A hyphenated or dotted segment (e.g. `:foo-bar`) now types as a literal,
  matching the runtime matcher, rather than as a required param. Diff-invisible:
  it is a type-shape change to the existing `RouteParams`.
- ⚠ **`engines.node` raised to `^22.18.0 || >=24.11.0`** on `hono-preact` and
  `@hono-preact/vite` (#115, Babel 8); the other packages stay `>=20`.
  Diff-invisible: it is a `package.json` engines change, not an export change.

## New features

### Realtime (WebSocket-first)

A first-class realtime layer rides one `/__sockets` transport with a Cloudflare
Durable Object backend, all inheriting the route tree's `use` guard chain:

- **Channels + live loaders** (`defineChannel`, `publish`) — server-to-client
  broadcast topics; a live loader streams updates into `loader.View`.
- **Sockets** (`defineSocket`, `useSocket`, `upgradeWebSocket`) — a typed duplex
  WebSocket with exponential-backoff reconnect, a bounded send queue, and a
  reactive `status`. The server `open(socket)` handler does not receive a Hono
  `Context`; a `data?: (c) => Data` factory seeds read-only `socket.data`, which
  keeps a handler portable to the Durable Object runtime where no live Context
  exists.
- **Rooms + presence** (`defineRoom`, `useRoom`) — a duplex socket bound to a
  typed channel plus a presence roster; server-mediated fan-out via
  `conn.broadcast`.

On Cloudflare Workers, all three run on the `HONO_PREACT_REALTIME` Durable
Object: rooms and pub/sub topics get one DO per channel key (on disjoint id
namespaces) and plain sockets get a fresh per-connection DO. Cross-isolate
live-loader pub/sub and `publish()` fan out through the topic's DO, so live
loaders update across isolates just as they do in one Node process.

Realtime option bags use the full word `Options` (`UseSocketOptions`,
`UseRoomOptions`, `ReconnectOptions`, `LiveLoaderOptions`), matching the existing
`UseActionOptions`/`UseOptimisticOptions`.

Guards on a realtime upgrade must use `deny()`, not `redirect()` (a WebSocket
handshake cannot follow an HTTP redirect; a `redirect()` is treated as a deny and
logs a warning). On Cloudflare, `conn.data`/`socket.data` is edge-seeded
read-only metadata; use `setPresence` for per-connection state that evolves.

### Standard Schema validation + accessible forms

- `getValidationIssues` and `defineAction({ input })`,
  `defineLoader({ searchSchema, paramsSchema })`, `<Form schema>` accept any
  Standard Schema validator. Action failures become `deny(422)`; loader failures
  throw to the error boundary.
- `useFieldErrors` / `<FieldError>` surface client and server field errors
  uniformly. New `useFieldErrorProps(name)` returns the ARIA props
  (`aria-invalid`, `aria-describedby` pointing at the `<FieldError>` id) to
  spread onto a field control, so form errors are programmatically associated for
  assistive technology by default.

### Headless Toast primitive (`hono-preact-ui`)

A Sonner-shaped imperative `toast()` + `<Toaster>`: top-layer Popover region,
swipe-to-dismiss, promise toasts, and a polite/assertive live-region announcer.

### Quality pass

The full internals backlog plus the server-API split landed: typed-routing
rigor, the `Serialize<T>` wire-mirror, live streams via `loader.View`'s
accumulating form, and the delayed loader fallback.

## New public exports

- **Realtime:** `defineChannel`/`Channel`/`Topic`, `publish`, `LiveLoaderOptions`,
  `defineSocket`/`SocketRef`/`SocketHandler`/`ServerSocket`, `useSocket` (with
  `SocketStatus`/`SocketCloseInfo`/`ReconnectOptions`/`UseSocketOptions`/`UseSocketResult`),
  `upgradeWebSocket`, `defineRoom`/`RoomRef`/`RoomHandler`/`RoomConnection`,
  `useRoom`/`UseRoomOptions`/`UseRoomResult`, `PresenceMember`.
- **Validation/forms:** `getValidationIssues`/`ValidationIssue`,
  `useFieldErrors`/`useFieldErrorProps`/`FieldError`/`FieldErrorsMap`, and the
  re-exported `StandardSchemaV1` type.
- **Types:** `Serialize<T>`, `AnyLoaderRef`, `StreamStatus`.
- **Cloudflare adapter:** `CloudflareAdapterOptions` (from
  `hono-preact/adapter-cloudflare`).
- **UI (`hono-preact-ui`):** the imperative `toast`, `<Toaster>` (`ToasterProps`),
  and the `Toast*` parts `ToastRoot`/`ToastTitle`/`ToastDescription`/`ToastClose`/
  `ToastAction` (each with its `*Props`), plus the supporting types `Toast`,
  `ToastOptions`, `ToastPosition`, `ToastRecord`, `ToastType`, `ToastActionData`.

## New entry points and dependencies

- New subpaths `hono-preact/server/internal/cloudflare` and
  `@hono-preact/server/internal/cloudflare` expose the realtime Durable Object
  glue for the generated Cloudflare entry.
- New runtime dependency `@standard-schema/spec` (type-only) on `iso` and the
  `hono-preact` umbrella. The `@hono-preact/server` package adds the Node
  WebSocket stack (`@hono/node-server`, `@hono/node-ws`, `ws`, `@types/ws`) and
  `@cloudflare/workers-types` as build/type-time **devDependencies**; consumers
  do not take them on transitively.
